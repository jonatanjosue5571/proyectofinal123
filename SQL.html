Lenguaje de SQL

SQL (Structured Query Language) es un lenguaje especializado que permite interactuar con bases de datos. Sus acciones de lenguaje pueden clasificarse en función de su propósito:

  1. Acción Representativa (Informativa) → Se usa para consultar información de la base de datos.
  2. Acción Expresiva → Expresa estados o cambios dentro de la base de datos.
  3. Acción Apelativa (Conativa) → Indica comandos que deben ejecutarse.
  4. Acción Fática → Verifica la conexión con la base de datos.

Funciones del Lenguaje SQL

La función principal del lenguaje SQL es referencial o informativa, ya que su objetivo es gestionar datos en bases de datos relacionales. Sin embargo, dependiendo del contexto, también cumple funciones como:

  • Control y gestión de datos → Permite definir, modificar, consultar y administrar datos.
  • Control de accesos y permisos → Gestiona usuarios y privilegios dentro de la base de datos.
  • Control de transacciones → Garantiza la integridad y consistencia de los datos.

Comandos y Ejemplos de SQL

SQL se divide en cinco categorías principales de comandos:

  1. CREATE: Crea bases de datos, tablas, índices o vistas.
  2. ALTER: Modifica las estructuras de una tabla, base de datos o índices.
  3. DROP: Elimina una tabla, base de datos o índices.

Ejemplos

  • SELECT * FROM empleados;
  • SELECT nombre, edad FROM empleados WHERE edad > 30;

Lenguaje DDl

El lenguaje de definición de datos (DDL) es un conjunto de instrucciones que permite crear, modificar y eliminar objetos de una base de datos.

Qué objetos se pueden crear con DDL

Tablas, Esquemas, Vistas, Secuencias, Catálogos, Índices, Variables, Máscaras, Permisos, Alias.


Funmciones del DDL

Permite eliminar, editar y añadir objetos de base de datos. Algunas de las sentencias DDL más comunes que se ejecutan incluyen CREATE , DROP , COMMENT , ALTER y otras. Los comandos DDL suelen ejecutarse en un navegador SQL o un procedimiento almacenado.Funciones del DDL

  • Crear tablas, secuencias, ubicaciones, alias, esquemas e índices
  • Modificar tablas existentes
  • Eliminar objetos que ya no se necesitan
  • Configurar el marco o estructura de una base de datos

Sus Comanndos y Ejemplos

Se pueden ejecutar en un explorador SQL o un procedimiento almacenado

Ejemplos de comandos DDL

  • CREATE SCHEMA para crear un esquema
  • CREATE TABLE para crear una tabla
  • DECLARE GLOBAL TEMPORARY TABLE para crear una tabla temporal
  • TRUNCATE TABLE para eliminar entradas de datos de una tabla

Lenguaje DML

Accuiones en el Lenguaje DML

El lenguaje DML en SQL permite la manipulación de los datos dentro de las tablas de una base de datos. Sus acciones de lenguaje incluyen:

Funciones y Ejemplos

  • • Gestión de datos → Permite insertar, modificar y eliminar información en la base de datos.
  • • Recuperación de información → Facilita la consulta de datos a través de SELECT.
  • • Interacción con los usuarios → Los datos pueden actualizarse dinámicamente según las necesidades.
  • Acción Expresiva → Inserta o modifica datos existentes en una tabla.
  • Acción Apelativa (Conativa) → Da órdenes que afectan los datos almacenados.

DML se enfoca en la manipulación de los datos almacenados en las tablas y no en la estructura de la base de datos.Sus Comandos:

SELECT Consultar datos en una tabla Este comando permite recuperar información de la base de datos.


Lenguaje DCL

DCL (Data Contro, Lenguaje de Control de Datos) 1. Acciones del Lenguaje en DCL,DCL en SQL se usa para gestionar los permisos y el acceso a la base de datos. Sus acciones principales incluyen: 1. Acción Apelativa (Conativa): Se usa para otorgar o revocar permisos a usuarios.

Funciones del DCL

DCL cumple principalmente con la función apelativa, ya que permite administrar los permisos de acceso y seguridad de la base de datos.Gestión de usuarios: Controla quién puede acceder a la base de datos y qué operaciones puede realizar.Seguridad de datos: Restringe el acceso a información sensible.Auditoría y control: Permite rastrear qué usuarios tienen qué permisos.

Su Comando

Comando DCL se usa para otorgar o revocar privilegios en la base de datos.A. GRANT,Otorgar permisos a un usuario.Este comando asigna permisos específicos a un usuario o rol.

  • Ejemplo: Dar permiso de lectura en una tabla a un usuario

Lenguaje TCL

TCL en SQL se usa para gestionar transacciones en una base de datos, asegurando que las operaciones sean ejecutadas de manera segura y consistente. Las acciones principales incluyen:1. Acción Representativa (Informativa): Se usa para verificar el estado de una transacción.

Su Funcion

TCL cumple principalmente con la función metalingüística y conativa, ya que permite controlar cómo se ejecutan y confirman las transacciones en la base de datos.

Sus Comandos

TCL se centra en la administración de transacciones dentro de la base de datos.A. COMMIT Confirmar una transacción,Este comando guarda permanentemente los cambios realizados en la base de datos.
